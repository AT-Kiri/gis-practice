# code-review 技能

## 用途

编码完成后执行代码审查，检查规范合规性、潜在 Bug、死代码等。

## 触发时机

- Phase 4 验收交付前
- 大功能开发完成后

## 审查清单

1. **规范合规** — 组件命名、文件结构、Git 提交格式
2. **潜在 Bug** — 未定义变量、空指针、异步错误未捕获
3. **死代码** — 未使用的 import、变量、组件 Props
4. **安全风险** — 密钥硬编码、凭证泄露
5. **SuperMap 相关** — iServer 地址使用相对路径、GeoJSON 图层清理
6. **Vue 规范** — 正确使用 Composition API、scoped 样式

## 输出

生成审查报告，按严重程度分类：

| 级别 | 说明 | 处理方式 |
|------|------|----------|
| P0 | 阻断性问题（Bug、安全） | 必须修复 |
| P1 | 规范性问题 | 建议修复 |
| P2 | 建议优化 | 可选 |

审查通过后可在 `project_rules.md` 中更新审查状态。
