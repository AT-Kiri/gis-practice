# Checklist: 多智能体协同 GIS 应急助手

> 验收检查清单，逐项核对通过方可验收

---

## 1. 功能验收（P0 核心）

### 1.1 对话界面
- [x] 右侧悬浮按钮，点击展开/收起聊天面板
- [x] 用户消息与 AI 消息样式区分明确
- [x] 输入框支持 Enter 发送和按钮发送
- [x] 空消息不发送
- [x] 清空对话功能正常
- [x] 对话历史可滚动查看

### 1.2 流式回答
- [x] AI 回答逐字流式显示（非一次性出现）
- [x] 思考中 / 加载状态指示清晰
- [x] 回答完成后状态正确切换
- [x] 网络异常时有错误提示

### 1.3 专题检索工具
- [x] 输入"朝阳区在哪"，Agent 正确调用 feature_search
- [x] 返回结果包含要素列表和 GeoJSON
- [x] 地图上自动标绘检索结果（橙色点/线/面）
- [x] 地图自动缩放到结果范围
- [x] 对话中显示搜索结果摘要
- [x] 空关键字 / 无结果时正确处理

### 1.4 空间查询工具
- [x] Agent 可通过空间查询获取指定范围内的要素
- [x] 查询结果正确渲染到地图
- [x] 结果数量和摘要信息正确显示

### 1.5 缓冲区分析工具
- [x] 指定位置和半径，Agent 正确调用 buffer_analysis
- [x] 缓冲区结果（面）正确渲染到地图
- [x] 结果摘要包含缓冲区基本信息
- [x] 半径为 0 或负数时正确报错

### 1.6 叠置分析工具
- [x] 指定两个数据集和操作类型，Agent 正确调用 overlay_analysis
- [x] 叠置结果正确渲染到地图
- [x] 结果要素数量正确显示

### 1.7 网络分析工具
- [x] 指定起点终点，Agent 正确调用 shortest_path
- [x] 路径线正确渲染到地图
- [x] 路径距离正确显示
- [x] 自动切换到长春底图（如果当前不在长春视图）
- [x] 服务区分析功能正常
- [x] 点数不足时正确报错

### 1.8 地图定位工具
- [x] fly_to_location 工具返回正确的中心点和缩放
- [x] 前端收到结果后地图正确飞行定位

### 1.9 工具调用可视化
- [x] 每个工具调用显示独立的卡片
- [x] 卡片有"进行中/完成/失败"三种状态
- [x] 进行中状态有加载动画
- [x] 完成状态有对勾标识
- [x] 失败状态有错误信息和红色标识
- [x] 工具执行顺序与任务计划一致

### 1.10 Agent 结果图层管理
- [x] Agent 生成的结果使用专属配色（橙色系）
- [x] 与用户手动操作的结果视觉上可区分
- [x] 清空对话时同步清除地图上的 Agent 结果
- [x] 多个工具结果叠加显示正常

## 2. 功能验收（P1 重要）

### 2.1 RAG 知识库
- [x] 系统启动时自动索引初始知识库文档
- [x] KnowledgeAgent 正确调用 rag_retrieval 工具
- [x] 检索结果按相关度排序
- [x] 回答基于检索到的知识生成
- [x] 知识库无相关内容时不编造
- [x] 文档上传接口可用
- [x] 向量检索接口可用

### 2.2 多智能体协同
- [x] Coordinator 正确识别用户意图
- [x] 意图分类输出符合 JSON Schema
- [x] Schema 校验失败时自动重试（最多 3 次）
- [x] 任务规划生成合理的多步任务
- [x] 下一个 Agent 由代码路由，不由 LLM 自由决定
- [x] 子 Agent 输出符合契约（固定字段）
- [x] 最多 5 步保护生效（MAX_STEPS=5）
- [x] 多步任务分步执行，每步结果依次出现
- [x] Coordinator 最终汇总生成综合回答（流式输出）
- [x] 双轨制路由：单一意图走单 Agent，mixed 走 Coordinator
- [x] 闲聊跳过意图分类直接流式回答
- [x] _build_step_context 注入前序 summary 和关键数据作为子 Agent 上下文

### 2.3 分步可视化
- [x] 多步任务中，每步结果实时标绘到地图
- [x] 前端显示当前进度（第 X / 共 Y 步）
- [x] 步骤说明文字同步显示
- [x] 最终所有结果叠加在地图上
- [x] 模拟数据用紫灰色系 + [模拟] 弹窗标签区分

### 2.4 应急场景演示
- [x] "朝阳区发生地震，评估灾情并规划救援路线"能走通完整流程
- [x] 至少包含 3 步以上操作（实际 5 步：定位→缓冲区→查医院→路径规划→知识检索）
- [x] 每步结果都能在地图上看到
- [x] 最终有综合报告输出
- [x] 应急救援场景 spatial_query 只查点要素（不再有线/面要素干扰）
- [x] 真实资源缺失时用 mock_nearby_resources 生成模拟数据 + 显式告知用户
