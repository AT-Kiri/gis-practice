# Spec: Agent 对话界面（Agent Chat）

> 模块：前端 AgentChatPanel 组件 + 后端 SSE 接口
> 职责：提供对话式交互界面，流式展示 AI 回答和工具调用过程

---

## 场景 1：聊天面板展开/收起

### Given-When-Then

#### 正常场景：展开面板
- **Given** 用户在主页，看到右侧悬浮的 Agent 按钮
- **When** 用户点击悬浮按钮
- **Then**
  - 聊天面板从右侧滑入
  - 面板宽度约 380px，高度约 80vh
  - 显示欢迎消息和快捷操作建议
  - 悬浮按钮变为收起按钮

#### 正常场景：收起面板
- **Given** 聊天面板已展开
- **When** 用户点击收起按钮或点击面板外区域
- **Then**
  - 面板滑出收起
  - 保留对话历史（当前会话内）
  - 显示悬浮按钮

---

## 场景 2：发送消息

### Given-When-Then

#### 正常场景：发送文本消息
- **Given** 用户在输入框输入"朝阳区在哪"
- **When** 用户点击发送按钮或按 Enter
- **Then**
  - 消息立即出现在对话列表中（用户气泡，右侧）
  - 输入框清空
  - 显示"AI 思考中..."加载状态
  - 发送请求到后端 `/api/agent/chat`

#### 边界场景：空消息
- **Given** 输入框为空或只有空格
- **When** 用户点击发送
- **Then**
  - 不发送请求
  - 输入框保持不变
  - （可选）轻微抖动提示

#### 边界场景：超长消息
- **Given** 用户输入超过 2000 字
- **When** 用户发送
- **Then**
  - 截断或提示"消息过长，请精简后发送"
  - 具体阈值：2000 字符

---

## 场景 3：流式回答展示

### Given-When-Then

#### 正常场景：纯文本流式回答
- **Given** 后端返回 SSE `text` 事件
- **When** 前端收到流式 token
- **Then**
  - AI 消息气泡中逐字追加内容
  - 光标闪烁效果（可选）
  - 消息列表自动滚动到底部

#### 正常场景：工具调用过程展示
- **Given** 后端返回 `tool_start` 事件
- **When** 前端收到事件
- **Then**
  - 在对话中插入"工具调用"卡片
  - 显示工具名称 + 加载动画
  - 卡片位于 AI 消息气泡内

#### 正常场景：工具完成结果展示
- **Given** 后端返回 `tool_result` 事件，含 geojson
- **When** 前端收到事件
- **Then**
  - 工具卡片状态变为"完成"（绿色对勾）
  - 显示结果摘要文字
  - **自动将 GeoJSON 渲染到地图上**
  - 地图自动缩放到结果范围（fitBounds）
  - 卡片上有"定位到地图"按钮（可选，点击 flyTo）

#### 异常场景：工具调用失败
- **Given** 工具返回 success=false
- **When** 前端收到 tool_result
- **Then**
  - 工具卡片状态变为"失败"（红色感叹号）
  - 显示错误信息
  - 不影响后续工具执行和回答

---

## 场景 4：对话历史管理

### Given-When-Then

#### 正常场景：多轮对话
- **Given** 用户已经问了几个问题
- **When** 继续对话
- **Then**
  - 所有历史消息按时间顺序排列
  - 用户消息右侧，AI 消息左侧
  - 可滚动查看历史
  - 历史消息中的地图结果保留在地图上

#### 正常场景：清空对话
- **Given** 用户点击"清空对话"按钮
- **When** 确认清空
- **Then**
  - 对话列表清空
  - 地图上所有 Agent 结果被清除
  - 会话 ID 重置

---

## 场景 5：快捷操作

### Given-When-Then

#### 正常场景：快捷问题推荐
- **Given** 对话刚开始或回答完成后
- **When** 面板底部显示快捷问题按钮
- **Then**
  - 显示 3-4 个推荐问题，如：
    - "朝阳区有哪些医院？"
    - "给我做一个 1km 缓冲区分析"
    - "从北京到天津的最短路径"
  - 点击按钮直接发送对应问题

## SSE 事件处理契约

### 事件类型与处理逻辑

| 事件类型 | 数据字段 | 前端处理 |
|---------|---------|---------|
| `agent_start` | agent_name | 显示思考中状态 |
| `tool_start` | tool_name, input | 插入工具卡片（加载中） |
| `tool_result` | tool_name, result, geojson? | 更新工具卡片 + 渲染 GeoJSON 到地图 |
| `agent_step` | agent_name, step_result | 更新进度（可选） |
| `text` | content | 追加到 AI 消息文本 |
| `agent_end` | final_answer | 标记消息完成，停止加载 |
| `error` | message | 显示错误消息 |

### GeoJSON 渲染规则
1. 每个工具结果创建独立的 Source/Layer
2. Source ID 格式：`agent-result-{toolName}-{timestamp}`
3. 样式：使用 Agent 主题色系（橙色/琥珀色），与用户手动操作结果区分
4. 结果保留直到用户清空对话或手动清除
5. 自动 fitBounds 到结果范围（padding: 50）
