## Context

专题检索模块允许用户通过关键字和行政级别过滤快速定位地物要素。构建在已完成的地图容器和空间查询基础之上。该模块与空间查询（模块2）互补——空间查询是"画范围找要素"，专题检索是"搜关键字找要素"。

## Goals / Non-Goals

**Goals:**
- 关键字模糊搜索（跨数据集，匹配 NAME 等关键字段）
- 按行政级别过滤（省级/县级/乡镇级）
- 结果分页展示 + 地图标绘
- 点击结果查看完整属性详情
- 搜索历史记录
- 一键清除

**Non-Goals:**
- 不实现空间+属性的联合查询（后续可扩展）
- 不实现高级查询语法（AND/OR 表达式）

## Decisions

### 1. 前端架构

```
FeatureSearch.vue
├── 搜索输入框（带历史下拉）
├── 行政级别过滤（Radio/Select）
├── 结果面板（分页列表）
├── 要素详情面板（属性表格）
└── 地图标绘层（独立 GeoJSON source + layers）
```

### 2. 后端查询策略

后端通过 `attributeFilter` 参数调用 iServer 数据服务的 `GET /datasets/{name}/features` 接口：

- 关键字搜索：`attributeFilter=NAME like '%关键词%'`
- 行政级别过滤：通过限定数据集来实现
  - 省级 → `Province_L`, `County_P`（省会级别）
  - 县级 → `County_P`
  - 乡镇级 → `Town_P`
- 全量搜索：所有数据集

### 3. 行政级别映射

| 级别 | 数据集 | 说明 |
|---|---|---|
| 省级 | Province_L, County_P(省会城市) | 省界、省会 |
| 县级 | County_P | 县级市 |
| 乡镇级 | Town_P | 乡镇 |

### 4. 结果展示

- **结果列表**：分页展示（每页10条），显示名称 + 数据集 + 类型标签
- **地图标绘**：与空间查询模块复用相同的标绘样式（蓝点/蓝线/半透明蓝面）
- **详情面板**：点击结果项后弹出，以 key-value 表格展示所有属性字段

### 5. 搜索历史

- 使用 localStorage 存储最近10条搜索记录
- 点击搜索框时显示历史下拉
- 点击历史项快速重新搜索

## Risks / Trade-offs

| 风险 | 缓解措施 |
|---|---|
| 关键字搜索在大量数据上性能慢 | iServer 原生支持 attributeFilter，可利用数据库索引 |
| 不同数据集中文名称字段不一致 | 统一检查 NAME/名称/地名 等多个字段 |
