# 数据大屏模块 - 行为规格

## 模块 1: 灾害程度分级地图 (DashboardMap)

### 场景 1.1：正常加载显示分级地图

**Given** 用户进入数据大屏页面  
**When** 地图加载完成，各县区 GeoJSON 数据就绪  
**Then** 京津冀各县区根据灾害等级渲染为对应的填充颜色：
- 等级 1（低风险）→ 绿色 `#52c41a`
- 等级 2（较低）→ 黄色 `#fadb14`
- 等级 3（中风险）→ 橙色 `#fa8c16`
- 等级 4（高风险）→ 红色 `#f5222d`
- 等级 5（极高）→ 深红 `#a8071a`

**And** 地图默认视角居中于京津冀区域（`center: [116.4, 39.9]`，`zoom: 8`）

### 场景 1.2：鼠标悬停显示 tooltip

**Given** 分级地图已加载  
**When** 用户将鼠标悬停到某个县区上  
**Then** 鼠标指针变为手型指针  
**And** 显示自定义 tooltip，内容包含：地区名称、灾害类型、所需救援方式、备注  
**And** tooltip 跟随鼠标移动  

### 场景 1.3：点击县区选中

**Given** 分级地图已加载  
**When** 用户点击某个县区  
**Then** 该县区边界高亮显示（如加粗描边）  
**And** 右侧详情面板切换显示为该县区的灾害详情数据  
**And** 若再次点击同一个县区，高亮取消，详情面板保持显示

### 场景 1.4：悬停/点击到无数据区域

**Given** 分级地图已加载  
**When** 用户悬停或点击到京津冀范围外的区域（如海洋）  
**Then** 不显示 tooltip，不触发点击事件，右侧面板保持当前状态不变

## 模块 2: 灾害详情面板 (DisasterDetailPanel)

### 场景 2.1：正常显示灾害详情

**Given** 用户已点击地图上的某个县区  
**When** 右侧详情面板打开  
**Then** 面板展示以下信息：地区名称、受灾类型、受灾等级（带颜色标签）、救援人员数量、受灾人员数量、备注文字  
**And** 面板标题为"XX县 - 灾害详情"

### 场景 2.2：联动分析按钮

**Given** 灾害详情面板正在显示  
**When** 用户点击"缓冲区分析"按钮  
**Then** 弹出 `BufferAnalysisModal` 弹窗，以选中县区中心为圆心进行缓冲区分析  
**And** 弹窗默认缓冲区半径为 5km

### 场景 2.3：未选中时面板状态

**Given** 用户第一次进入数据大屏页面，未点击任何县区  
**Then** 右侧详情面板显示提示信息："请点击地图上的县区查看灾害详情"

---

## 模块 3: 缓冲区联动分析 (BufferAnalysisModal)

### 场景 3.1：正常加载缓冲区分析结果

**Given** BufferAnalysisModal 已打开  
**When** 缓冲区分析完成  
**Then** 弹窗展示以下内容：
- 缓冲区半径（默认 5km，可通过滑块调节 1-50km）
- 附近救援人员列表：地点名称 + 人员数量
- 可分配物资列表：地点名称 + 物资类型/数量

**And** 地图上以半透明圆圈渲染缓冲区  
**And** 救援点和物资点以不同颜色的标记点显示在地图上

### 场景 3.2：调整缓冲区半径

**Given** BufferAnalysisModal 已打开，且已显示分析结果  
**When** 用户拖动半径滑块从 5km 调整到 10km  
**Then** 缓冲区圆圈自动更新为 10km 半径  
**And** 救援人员和物资列表随半径增大而更新（可能包含更多远距离资源）

### 场景 3.3：缓冲区半径设为 1km

**Given** BufferAnalysisModal 已打开  
**When** 用户将半径滑块拖动到最小值 1km  
**Then** 缓冲区圆圈缩小至 1km  
**And** 资源列表更新，如果 1km 范围内无资源则显示"该范围内未找到救援资源"

### 场景 3.4：弹窗关闭

**Given** BufferAnalysisModal 已打开  
**When** 用户点击弹窗右上角关闭按钮或点击"取消"  
**Then** 弹窗关闭  
**And** 地图上的缓冲区圆圈和标记点被清除

## 模块 4: 路径规划 (RoutePlanningModal)

### 场景 4.1：正常路径规划

**Given** 用户已看到缓冲区分析结果中的救援/物资点  
**When** 用户点击某个资源点旁的"救援"按钮  
**Then** 打开 RoutePlanningModal 弹窗  
**And** 调用 iServer 网络分析服务计算从该救援点到受灾县区的最优路径  
**And** 弹窗展示：总距离、预计所需时间  
**And** 地图上高亮显示最优路径

### 场景 4.2：路径规划失败降级

**Given** RoutePlanningModal 已打开  
**When** iServer 网络分析服务返回空或无可用路网  
**Then** 显示提示信息"路网数据不可用，已显示直线路径"  
**And** 在地图上以虚线渲染起点到终点的直线  
**And** 弹窗依然可正常查看和关闭

### 场景 4.3：多资源点可选

**Given** 缓冲区分析结果列表中包含多个救援点  
**When** 用户点击不同的救援点旁的"救援"按钮  
**Then** RoutePlanningModal 更新为对应救援点的路径规划结果  
**And** 地图上移除旧的路径渲染，更新为新的路径

## 模块 5: 气象监控面板 (WeatherPanel)

### 场景 5.1：正常显示气象数据

**Given** 气象面板已加载  
**When** 数据就绪  
**Then** 面板以表格/卡片形式展示：地区、风险等级（带颜色标签）、大风强度（级）、降雨深度（mm）、地震强度（级）、受灾类型、受困人员数量、可用物资数量  

### 场景 5.2：阈值预警标红

**Given** 气象数据已加载  
**When** 某行数据的某个指标超过阈值：
- 降雨深度 > 100mm/h
- 大风强度 >= 10级
- 地震强度 >= 5.0级  
**Then** 超过阈值的数值以红色文字显示，并在该行添加红色边框或闪烁背景提醒

### 场景 5.3：显示前后几天天气

**Given** 气象面板已加载  
**When** 用户查看某地区的数据  
**Then** 面板底部显示该地区前 3 天 + 后 3 天的天气预报图表：
- 每天显示：日期、天气图标、最高/最低温度、降水概率

### 场景 5.4：表格排序

**Given** 气象面板显示多地区数据  
**When** 用户点击表头（如"风险等级"或"降雨深度"）  
**Then** 表格按该列排序（升序/降序切换）

---

## 模块 6: 数据入口 (导航栏)

### 场景 6.1：导航到数据大屏

**Given** 用户在任意页面  
**When** 点击顶部导航栏的"数据大屏"按钮  
**Then** 路由跳转到 `/data-dashboard`  
**And** 导航按钮高亮显示

### 场景 6.2：从数据大屏返回

**Given** 用户在数据大屏页面  
**When** 点击顶部导航栏的"二维地图"或"三维洪水模拟"  
**Then** 路由跳转到对应页面  
**And** 对应导航按钮高亮显示
