# 数据大屏模块 - 需求论证

## Why（业务背景）

京津冀城市综合防灾应急管理 GIS 项目目前已实现基本地图浏览、空间查询、缓冲区分析、网络分析等功能。为提升灾害监控和应急指挥的可视化能力，需要开发一个**数据大屏模块**，用于：

1. **综合态势感知**：以颜色分级展示京津冀地区各县的灾害程度，实现快速风险识别
2. **实时监控预警**：展示气象灾害（洪涝、大风、地震等）实时数据，超过阈值自动标红预警
3. **救援调度联动**：通过缓冲区分析和路径规划，快速匹配受灾区域与救援资源
4. **决策支持**：整合灾情、资源、天气等多维度数据，为应急指挥提供数据支撑

## What（变更内容）

新增一个独立的数据大屏页面，包含以下核心功能：

### 1. 灾害程度分级地图
- 使用颜色分级（绿→黄→橙→红）展示京津冀地区各县的灾害程度
- 鼠标悬停显示：地区名称、灾害类型、所需救援方式、备注等信息
- 点击区域弹出详情窗口

### 2. 灾害详情与联动分析
- **详情窗口**（右上角）：显示地区、受灾类型、受灾等级、救援人员数量、受灾人员数量、备注等
- **联动分析**：点击联动按钮，以受灾区域为中心进行缓冲区分析，显示附近的救援人员地点/数量、可分配物资地点/数量
- **缓冲区设置**：支持手动调整缓冲区半径
- **路径规划**：在救援/物资弹窗上提供救援按钮，自动计算从空闲救援点到受灾区的最优路径

### 3. 气象灾害监控面板
- 展示部分地区洪涝、大风、地震等实时数据
- 显示风险等级、大风强度、降雨深度、地震强度等指标
- 超过阈值自动标红预警
- 展示受灾类型、救援人员数量、受困人员数量、可用物资数量
- 显示该地区前几天的天气历史和后几天的天气预报

### 4. 数据入口
- 在顶部导航栏新增"数据大屏"入口，与"二维地图"、"三维洪水模拟"平行

## Capabilities（新增/修改能力）

| 能力 | 类型 | 说明 |
|------|------|------|
| 数据大屏页面 | 新增 | 独立的 Vue 页面组件，半屏面板式布局 |
| 灾害分级渲染 | 新增 | 基于行政区划 GeoJSON 的颜色分级展示 |
| 灾害详情弹窗 | 新增 | 点击区域弹出灾害详细信息 |
| 缓冲区联动分析 | 新增 | 基于 SuperMap 缓冲区分析的救援资源查询 |
| 最优路径规划 | 新增 | 从救援点到受灾区的最优路径计算 |
| 气象监控面板 | 新增 | 实时气象数据展示与阈值预警 |
| 模拟数据生成 | 新增 | 前端模拟灾情、救援资源、气象等数据 |
| 顶部导航入口 | 修改 | 在现有导航栏新增"数据大屏"入口 |

## Impact（影响范围）

### 影响模块
| 模块 | 影响类型 | 说明 |
|------|----------|------|
| `frontend/src/views/` | 新增 | 新增 `DataDashboard.vue` 页面 |
| `frontend/src/components/` | 新增 | 新增大屏相关子组件（灾害详情、联动分析、气象面板等） |
| `frontend/src/router/` | 修改 | 新增数据大屏路由 |
| `frontend/src/App.vue` 或导航组件 | 修改 | 新增顶部导航入口 |
| `frontend/src/utils/` | 新增 | 新增模拟数据生成工具 |
| `backend/` | 无影响 | 本模块使用模拟数据 + iServer 现有服务，不新增后端接口 |

### 技术依赖
- `@supermap/iclient-mapboxgl`：用于缓冲区分析、路径规划
- MapboxGL：地图渲染、交互
- Ant Design Vue：UI 组件（弹窗、滑块、卡片等）

### 数据依赖
- **iServer 服务**：京津冀行政区划 GeoJSON（已有）
- **模拟数据**：灾情数据、救援资源、气象数据（需前端生成）
